Interactive Attack Solver

01
Recover Nonce kk = (h₁ − h₂) × (s₁ − s₂)⁻¹ mod n
▶ Open

From the two ECDSA equations, subtracting eliminates the private key d and lets us solve for k directly.

// MATHEMATICAL DERIVATION
s₁ = k⁻¹(h₁ + r·d) mod n
s₂ = k⁻¹(h₂ + r·d) mod n
→ subtract:
s₁ − s₂ = k⁻¹(h₁ − h₂) mod n
→ solve for k:
k = (h₁ − h₂) · (s₁ − s₂)⁻¹ mod n
02
Recover Private Key dd = (s₁·k − h₁) · r⁻¹ mod n
🔒

Now that k is known, rearrange the ECDSA signing equation to solve for the private key d.

// ISOLATE d
s₁ = k⁻¹(h₁ + r·d) mod n
→ rearrange:
s₁·k = h₁ + r·d mod n
r·d = s₁·k − h₁ mod n
d = (s₁·k − h₁) · r⁻¹ mod n
03
Verify Against Both SignaturesConfirm d is correct: recompute s₁ and s₂
🔒

Sanity-check: use k and d to recompute s₁ and s₂ from scratch. Both must match exactly.

Background

ECDSA's security depends entirely on generating a fresh random nonce k for every signature. If k is ever reused, the private key can be recovered with simple algebra — no brute force, no curve math.


This vulnerability is what allowed the PS3's signing key to be extracted in 2010, and has broken countless cryptocurrency wallets.

ECDSA Signing

1
Choose random nonce k ∈ [1, n-1]
2
Compute R = k·G (point on curve)
3
r = R.x mod n
4
s = k⁻¹(h + r·d) mod n
⚠
If same k used twice → private key d is recoverable

Why Nonce Reuse Breaks It

s₁ = k⁻¹(h₁ + r·d) mod n
s₂ = k⁻¹(h₂ + r·d) mod n
↓ subtract
s₁ − s₂ = k⁻¹(h₁ − h₂)
↓ solve for k, then d
k and d both leak!
Two equations, two unknowns (k and d). The system is solvable with basic modular arithmetic — no elliptic curve operations needed.

Real-World Incidents

🎮
PlayStation 3 (2010)
Sony used a static k. GeoHot extracted the signing key and released it publicly.
₿
Bitcoin wallets (2013)
Android's SecureRandom weakness caused k reuse in Bitcoin wallets. Millions lost.
🔑
TLS certificates
Repeated nonces in ECDSA-signed TLS sessions have led to private key disclosure.

# Step 2: recover private key d

assert (k_inv * (h1 + r*d)) % n == s1

Hints

💡 HINT 1−30 pts▼ Reveal
The r-values are identical: r₁ = r₂. In ECDSA, r = (k·G).x mod n. Identical r means identical k. Write out both signing equations and subtract them — k cancels d, letting you solve for k directly.
💡 HINT 2−60 pts▼ Reveal
k = (h₁ − h₂) × (s₁ − s₂)⁻¹ mod n. Once you have k, recover d: d = (s₁·k − h₁) × r⁻¹ mod n. In Python 3.8+: pow(x, -1, n) computes modular inverse.

The private key d is a 77-digit decimal number. Always reduce intermediate results mod n. Verify your answer by checking that k⁻¹·(h₁ + r·d) mod n == s₁ and the same for s₂.
